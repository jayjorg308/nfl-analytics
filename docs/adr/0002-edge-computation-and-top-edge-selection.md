# Edge computation and top-edge selection

Matchup edge is `offensiveEpaPerPlay + defensiveEpaPerPlay` (addition, not subtraction) because defensive EPA is stored under the convention "what the defence allows per play" — strong defences have negative values, weak defences have positive values. Positive edge favours offence, negative favours defence. Worked example: SEA pass O (+0.12) + NE pass D (+0.04 allowed) = +0.16 (offensive edge); NE pass O (+0.05) + SEA pass D (−0.18 allowed) = −0.13 (defensive edge).

The slate dashboard's compact card highlights one "top edge" per game, selected by **max absolute value** across the four matchup edges (pass/rush × home-offence/away-offence). Max absolute, not max positive, because a dominant defensive matchup is just as research-relevant as a dominant offensive one (drives prop unders, totals leans, fading the smothered side); the naive "max positive" default would hide every "this team gets shut down" story. Tiebreaker when two edges are within ~0.01: prefer the matchup whose underlying team-level EPA values are more extreme, so the highlighted edge stays analytically meaningful rather than arithmetically maximal.

When the max absolute edge falls below ~0.05, the card renders "Even matchup — no clear edge" rather than highlighting a near-zero advantage. The 0.05 threshold is provisional and tunable once real-game data is available. The templated phrasing always leads with the dominant side of the matchup ("SEA's pass defence … vs NE's pass attack") rather than a fixed home/away order, so the sentence scans as the actual story.
