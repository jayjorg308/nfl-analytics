──────────────────────────────────────────────────────────────
REFERENCE / STATIC DATA
──────────────────────────────────────────────────────────────

TABLE: season
  - id (PK)
  - year                        e.g., 2026
  - startDate                   first regular season game
  - endDate                     Super Bowl date
  - currentWeek                 cached for performance
  - isComplete                  boolean

TABLE: stadium
  - id (PK)
  - name                        e.g., "Allegiant Stadium"
  - city
  - stateOrCountry              "NV" or "United Kingdom"
  - latitude
  - longitude
  - capacity
  - isDome                      boolean
  - hasRetractableRoof          boolean
  - surfaceType                 "grass" / "field turf" / "monofilament"
  - elevationFeet               relevant for Denver, Mexico City
  - isPrimaryHomeStadium        boolean (false for neutral-site-only venues)
  - homeTeamId                  FK -> team (nullable for neutral-site-only)

TABLE: team
  - id (PK)
  - name                        e.g., "Seattle Seahawks"
  - abbreviation                e.g., "SEA"
  - conference                  AFC / NFC
  - division                    e.g., "NFC West"
  - homeStadiumId               FK -> stadium
  - logoUrl
  - primaryColor                hex e.g., "#002244"
  - secondaryColor              hex e.g., "#69BE28"

TABLE: player
  - id (PK)
  - name
  - position                    QB / RB / WR / TE / etc.
  - jerseyNumber
  - heightInches
  - weightLbs
  - dateOfBirth
  - rookieYear
  - headshotUrl

TABLE: playerTeamMembership
  - id (PK)
  - playerId                    FK -> player
  - teamId                      FK -> team
  - startDate
  - endDate                     nullable (null = current)
  -- Handles mid-season trades. A player has one active membership
  -- at a time but multiple over a career.

──────────────────────────────────────────────────────────────
GAMES AND RESULTS
──────────────────────────────────────────────────────────────

TABLE: game
  - id (PK)
  - seasonId                    FK -> season
  - week                        1-18 regular season, plus playoff weeks
  - gameType                    regular / wildcard / divisional / conference / superbowl
  - homeTeamId                  FK -> team
  - awayTeamId                  FK -> team
  - stadiumId                   FK -> stadium
  - gameDateTime
  - isNeutralSite               boolean
  - isInternational             boolean
  - homeScore                   nullable until played
  - awayScore                   nullable until played
  - status                      scheduled / in-progress / final
  -- Weather (relevant for outdoor games):
  - temperature                 nullable
  - windMph                     nullable
  - precipitationChance         nullable
  - weatherCondition            e.g., "clear" / "snow" / "rain"

──────────────────────────────────────────────────────────────
PLAY-BY-PLAY (THE FOUNDATION OF EPA AND ADVANCED METRICS)
──────────────────────────────────────────────────────────────

TABLE: play
  - id (PK)
  - gameId                      FK -> game
  - quarter                     1-4, 5+ for OT
  - timeRemainingSeconds        seconds remaining in quarter
  - gameSecondsElapsed          for cross-quarter calculations
  - possessionTeamId            FK -> team (offense)
  - defensiveTeamId             FK -> team (defense)
  - down                        1-4, nullable for special situations
  - distance                    yards to first down
  - yardlineSide                "own" / "opp" / "50"
  - yardlineNumber              0-50
  - yardline100                 0-100, normalized (0 = own endzone, 100 = opp endzone)
  -- Score state before this play:
  - scoreOffense
  - scoreDefense
  -- Play type and outcome:
  - playType                    pass / run / punt / field_goal / kickoff / etc.
  - yardsGained                 nullable
  - isTouchdown                 boolean
  - isFieldGoal                 boolean
  - isFieldGoalSuccessful       boolean
  - isPunt                      boolean
  - isTurnover                  boolean
  - turnoverType                interception / fumble / nullable
  - isPenalty                   boolean
  - penaltyYards                nullable
  -- Players involved:
  - passerId                    FK -> player, nullable
  - rusherId                    FK -> player, nullable
  - receiverId                  FK -> player, nullable
  - tacklerId                   FK -> player, nullable (could expand to multi-player table later)
  -- EPA values (from nflfastR or your own computation):
  - expectedPointsBefore        EP at start of play
  - expectedPointsAfter         EP at end of play
  - epa                         the change (epAfter - epBefore)
  - successRate                 boolean, was this a "successful" play
  - isExplosive                 boolean, 20+ yard pass or 10+ yard run
  -- Pass-specific:
  - passLength                  short / deep / nullable
  - airYards                    nullable
  - yardsAfterCatch             nullable
  - isComplete                  nullable boolean
  -- Run-specific:
  - rushDirection               left / middle / right / nullable

  - gameId                      FK -> game

═══════════════════════════════════════════════════════════════
KEY DESIGN PRINCIPLES BAKED IN
═══════════════════════════════════════════════════════════════

1. TEMPORAL CORRECTNESS
   - teamWeekStats snapshots every week (no "current value only")
   - ELO rating snapshotted alongside other weekly metrics
   - propLineSnapshot and gameOddsSnapshot for line history
   - playerInjuryStatus tracks status changes over time
   - opponentDefenseRank* denormalized onto playerGame at ingestion

2. INDEPENDENT ENTITIES IN SEPARATE TABLES
   - season, stadium, team, player are independent reference data
   - Stadiums separate from teams (handles neutral sites, relocations)
   - Player team membership separate from player (handles trades)
   - Injuries separate from players (handles multiple injuries per season)
   - Injury status separate from injury (tracks weekly updates)

3. DENORMALIZED WHERE QUERY PATTERNS DEMAND IT
   - Opponent defense ranks on playerGame (for fast splits queries)
   - Weather on game (always needed alongside game data)
   - Season aggregates pre-computed in playerSeasonStats

4. STRUCTURAL SYMMETRY
   - Props and game odds follow the same pattern:
     parent record + snapshot table for line history
   - Makes ingestion and UI code consistent

5. PLAY-BY-PLAY AS FOUNDATION
   - Raw play data stored in postgres for direct querying
   - Aggregates pre-computed for UI performance
   - Research questions answerable via SQL against your own DB

═══════════════════════════════════════════════════════════════
INGESTION ORDER (CHICKEN-AND-EGG MANAGEMENT)
═══════════════════════════════════════════════════════════════

Weekly ingestion sequence after games complete:

1. Update game records with final scores
2. Insert play-by-play data into `play` table
3. Compute and update teamWeekStats:
   - EPA aggregates from plays
   - ELO recalculation (depends on prior week's ELO + this week's results)
   - SOS update
   - Traditional stat aggregates
4. Update playerGame records:
   - Per-player per-game stats from box scores
   - Denormalize opponentDefenseRank* from teamWeekStats (uses values
     computed in step 3 — must happen after)
5. Update playerSeasonStats:
   - Roll up player game stats into season aggregates
6. Update injury reports
7. Update prop/odds actualValue and actualResult fields for completed games

Continuous (multiple times per week):

- Snapshot prop lines from The Odds API
- Snapshot game odds from The Odds API
- Update upcoming game weather forecasts
- Update injury status reports
